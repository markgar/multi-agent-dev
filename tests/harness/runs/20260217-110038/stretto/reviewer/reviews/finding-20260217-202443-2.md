## Problem
Controller validation checks required/URL format but does not enforce persistence limits (`Title` 200, `Url` 2000, `DisplayName` 200, `FileName` 260, `ContentType` 200, `StoragePath` 1000). Oversized request values pass controller validation and fail later during persistence.

## Why it matters
Invalid user input should produce deterministic 4xx validation responses. Deferring this to the data layer causes avoidable 500s and inconsistent API behavior under normal malformed input.

## Suggested fix
Mirror persistence constraints in API validation and return `ValidationProblem` before saving.

Example additions:
```csharp
if (trimmedTitle.Length > 200)
{
    errors["title"] = ["Title must be 200 characters or fewer."];
}

if (trimmedUrl.Length > 2000)
{
    errors["url"] = ["Url must be 2000 characters or fewer."];
}

if (trimmedDisplayName.Length > 200)
{
    return ValidationProblem(new ValidationProblemDetails(new Dictionary<string, string[]>
    {
        ["displayName"] = ["DisplayName must be 200 characters or fewer."],
    }));
}
```
